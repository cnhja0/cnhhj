Sumário Executivo

O CNHhj está pronto no nível de código: o aplicativo do Instrutor tem todas as 16 telas planejadas implementadas, backend modelado, design system aplicado, e CI/CD configurado para gerar APKs na nuvem. Em horas estaremos com o primeiro APK instalável no celular.

O que é o CNHhj

Plataforma mobile que conecta instrutores autônomos de aulas práticas de CNH diretamente aos alunos em processo de habilitação. Hoje, esse público depende de autoescolas ou indicações boca-a-boca, sem visibilidade de preço, disponibilidade ou reputação dos instrutores. O CNHhj resolve isso.

Decisões-chave tomadas

  • Tecnologia mobile: Flutter — um único código serve iOS e Android (economia de ~50% no desenvolvimento mobile vs. apps nativos separados)
  • Backend: Supabase — banco PostgreSQL + autenticação + storage gerenciados; sem necessidade de DevOps próprio nessa fase
  • Foco geográfico: Guarulhos/SP — onde já temos contato local; baixo risco de escalar prematuramente
  • MVP gratuito: sem cobrança de comissão nem assinatura na primeira versão, para focar em validar a demanda antes de monetizar
  • Pagamento via PIX direto: no MVP, sem integração de gateway — reduz complexidade e custo de transação a zero
  • Build na nuvem (Codemagic): não precisamos de Mac nem instalação local de Flutter para gerar APKs Android

O que o app do Instrutor faz hoje

  • Cadastro completo em 7 passos (dados pessoais, veículo, fotos, documentos)
  • Login com e-mail/senha ou Google
  • Configuração de área de atuação, valor por aula e horários disponíveis
  • Pré-visualização do card que o aluno vai ver
  • Recebimento de solicitações de aulas com aceitar/recusar
  • Calendário de aulas confirmadas
  • Chat em tempo real com alunos (com opção de bloquear)
  • Avaliações recebidas (1 a 5 estrelas + comentário)

Modelo de negócio (futuro)

Não cobramos nada no MVP, mas o modelo já está validado conceitualmente:

  • Comissão de 20% sobre cada aula contratada via app (instrutor define R$100, aluno paga R$120)
  • Plano VIP — R$ 19,90/mês para os primeiros 100 instrutores (depois R$ 69,90)
  • Patrocínio: instrutores pagam para aparecer no topo da busca
  • Comissão sobre exames (fase 2, quando entrarem as clínicas) — 20% sobre exames médicos e psicotécnicos

Projeção otimista para Guarulhos sozinha: ~3.158 CNHs emitidas em janeiro de 2026. Com taxa de conversão de 30% e ticket médio de R$ 100, isso representa R$ 18.948/mês em comissão apenas no município inicial. São Paulo capital, próximo passo de expansão, multiplica esse número por uma ordem de magnitude.

05 · Tecnologia

Stack Técnica

Stack escolhida para minimizar custo e tempo de desenvolvimento no MVP, sem comprometer escalabilidade futura:

CamadaTecnologiaPor quê
App mobile Flutter iOS + Android com mesmo código. Comunidade gigantesca. Mantido pelo Google.
Estado da UI Riverpod Padrão moderno para Flutter, testável, sem boilerplate
Navegação go_router Rotas declarativas, suporta deep links (importante para futuro)
Backend Supabase PostgreSQL gerenciado + Auth + Storage + Realtime. Free tier generoso. Sem DevOps.
Build/CI Codemagic Gera APKs Android na nuvem. Não precisa de máquina dedicada nem Mac.
Repo GitHub Padrão de mercado, integrações com tudo, gratuito para repos privados

Arquitetura interna do app

O código está organizado por feature de produto (não por camada técnica). Cada feature reúne UI, lógica e estado. Padrões aplicados:

  • Repository Pattern: a UI nunca chama o backend diretamente — sempre via uma interface abstrata. Permite trocar mock por Supabase real sem reescrever telas.
  • Modo dual mock/real: hoje rodando com dados em memória. Com uma única variável de ambiente (APP_MODE=supabase), passa a usar o banco real.
  • Design System próprio: 20 componentes (botões, inputs, cards, modais) com a marca CNHhj. Toda nova tela é montada combinando esses tijolos — consistência visual garantida.
  • Row Level Security: autorização garantida no PostgreSQL, não só no app. Mesmo um cliente comprometido não acessa dados de outro usuário.

Banco de dados — 9 tabelas modeladas

Schema completo no arquivo backend/supabase/migrations/0001_initial_schema.sql:

  • profiles — identidade comum
  • instructors — dados do instrutor
  • students — dados do aluno (fase 2)
  • instructor_weekly_availability — agenda
  • bookings — aulas agendadas
  • conversations — threads de chat
  • messages — mensagens individuais
  • chat_blocks — bloqueios entre usuários
  • reviews — avaliações bidirecionais
06 · Produtividade

Aceleradores Instalados

O projeto usa Claude Code como ambiente de desenvolvimento — IA especializada que conhece nossa stack e padrões. Instalamos 16 skills especializadas em Flutter que aceleram tarefas futuras:

Skills de qualidade

  • flutter-tester — escreve testes unitários, de widget e de integração seguindo padrões Riverpod e Given-When-Then
  • flutter-code-reviewer — agente que revisa código antes de cada commit
  • flutter-dart-specialist — agente para problemas avançados de Dart

Skills de workflow

  • flutter-brainstorming / planning / executing — quebra features grandes em etapas verificáveis
  • flutter-verification / debugging — valida build e testa
  • flutter-parallel-agents — coordena múltiplos agentes para tarefas paralelas

Comandos slash

  • /brainstorm — explora ideias de feature antes de codar
  • /plan — produz plano técnico camada-por-camada
  • /execute — roda o plano com verificação automática
  • /init — inicializa novo projeto Flutter

Por que isso importa

Cada uma dessas skills representa horas economizadas em pesquisa e padrões. Ao invés de gastar tempo decidindo "como testar um Riverpod controller", a IA já sabe. Ao invés de descobrir "como debugar um erro de plataforma no Android", o agente especialista já tem o playbook. Estamos rodando com produtividade equivalente a um time sênior de Flutter, com custo de uma assinatura mensal de IA.

09 · Riscos

Riscos e Como Estamos Mitigando

RiscoProbabilidade / ImpactoMitigação
Problema do "ovo e galinha" do marketplace — sem instrutores, alunos não usam; sem alunos, instrutores não cadastram Alta / Alto Começar pelo lado da oferta (instrutores) com selo "Parceiro Fundador". Lançar app do aluno só depois de ter 5+ instrutores ativos.
Cadastro de instrutor sem validação no MVP permitiria fraudes Média / Médio No MVP, gratuidade desincentiva fraude (não há ganho financeiro). Aprovação manual entra junto com a fase paga (já preparada no código).
Aluno e instrutor combinam por fora do app — perdem comissão Alta / Alto (fase paga) Hoje não é um problema (MVP gratuito). Para a fase paga: incentivar permanência no app via chat in-app, comprovantes, avaliações. Termos de uso vão proibir transações fora.
Concorrência — empresas como Doutor Multas, AutoEscola Pro Média / Médio Foco geográfico (Guarulhos), nicho (instrutores autônomos vs. autoescolas), e simplicidade do app. Identidade forte (CNHhj) facilita boca-a-boca.
Custo de aquisição de aluno mais alto que LTV Média / Alto Validar com tráfego orgânico (Instagram, indicação) antes de pagar mídia. Boca-a-boca em Guarulhos onde temos contato local.
Compliance / regulação — DETRAN pode exigir validação formal de instrutores Baixa / Médio Termos de uso obrigam o instrutor a declarar credenciamento válido. Sistema de avaliação age como controle social.
"O maior risco aqui não é técnico — é de mercado. O código está pronto. O que precisa ser provado agora é se conseguimos atrair os primeiros 5 instrutores em Guarulhos."
